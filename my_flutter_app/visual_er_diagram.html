Key Relationships

  • User Inheritance: USER → CUSTOMER/CONSUMER (role-based)
  • Order Flow: CUSTOMER → ORDER → ORDER_ITEM → SERVICE
  • Service Management: CONSUMER → SERVICE (offerings)
  • Cart System: CUSTOMER → CART_ITEM → ORDER (checkout)
  • Delivery: ORDER → ADDRESS (delivery location)
  • Notifications: USER → NOTIFICATION (system messages)
  • Analytics: CONSUMER → ANALYTICS (business metrics)

Business Rules

  • Each USER can be either CUSTOMER or CONSUMER (role field)
  • CUSTOMER can have multiple ADDRESS entries
  • ORDER links CUSTOMER, CONSUMER, and delivery ADDRESS
  • ORDER contains multiple ORDER_ITEM entries
  • CART_ITEM are temporary until converted to ORDER
  • SERVICE pricing includes unit types (per lb, per item, per load)
  • ORDER status tracks: pending → in-progress → ready → completed
